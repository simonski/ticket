# Code of Conduct

## Our standards

We want the `ticket` community to be welcoming, constructive, and useful for
contributors, users, and maintainers.

Examples of behavior that we expect:

- be respectful and assume good intent
- give actionable, technical feedback
- disagree on ideas without attacking people
- be patient with questions from new contributors
- focus discussions on improving the project

Examples of unacceptable behavior:

- harassment, discrimination, or personal attacks
- trolling, intimidation, or deliberately disruptive behavior
- publishing private information without permission
- abusive language or hostile review comments
- repeated off-topic arguments after maintainers have made a decision

## Scope

This Code of Conduct applies to project spaces including issues, pull requests,
code reviews, discussions, and other public or private interactions related to
this repository.

## Reporting

If you experience or witness unacceptable behavior, open a private security
advisory or contact the maintainer through GitHub so the report can be handled
discreetly.

## Enforcement

Maintainers may remove comments, close discussions, reject contributions, or
temporarily or permanently restrict participation from people who violate this
Code of Conduct.
