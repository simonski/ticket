# UX Review

**Score: 68/100** (was 64)

## What is being assessed
CLI UX (help text, error messages, output formatting, destructive operation safety, discoverability, colour usage) and web UI UX (form validation, loading states, error display, mobile responsiveness, dark mode, keyboard navigation, accessibility, confirmation dialogs, success feedback).

## Methodology
Reviewed `cmd/ticket/help.go`, `cmd/ticket/printer.go`, `cmd/ticket/cmd_ticket.go`, `cmd/ticket/cmd_project.go`, and `web/static/index.html` (6080 lines) at v0.1.737. Checked for ARIA attributes, media queries, confirmation dialogs, loading state patterns, and colour-only state differentiation.

## Findings

### Passing checks
- CLI help system: hierarchical help with USAGE, DETAILS, EXAMPLE, ENVIRONMENT sections (`help.go`)
- Terminal-aware output: `term.GetSize()` with min-width clamping (`printer.go:383-392`)
- Consistent Unicode box-drawing tree rendering in `tk ls`
- Semantic colour palette: green=active, red=fail, gray=success, white=idle (`printer.go`)
- `--no-color` flag respected globally (`printer.go:271`)
- `tk rm` has two-step confirmation with a generated token (`cmd_ticket.go:2263-2286`) — fixed since last assessment
- Project deletion has two-step confirmation with token (`cmd_project.go`)
- Web UI: ticket delete now requires `uiConfirm()` before DELETE call (`index.html:5657`) — fixed since last assessment
- Web UI: WebSocket health monitoring with automatic reconnect polling (`index.html:4490`)
- Web UI: Viewport meta tag present — `width=device-width, initial-scale=1` (`index.html:5`)
- Web UI: Single responsive breakpoint at 900px collapses left panel (`index.html:308-312`)
- Web UI: Extensive keyboard navigation (arrow keys, Tab, E/D/U/S/P shortcuts) (`index.html:5942-6072`)
- Web UI: Semantic HTML elements: `<header>`, `<main>`, `<aside>`, `<nav>`, `<section>`
- Web UI: ARIA attributes: `role="dialog"`, `aria-modal="true"`, `aria-expanded`, `aria-label` on nav/modals (`index.html:1539, 962, 1680`)
- Web UI: `uiConfirm()` used for all major destructive operations (delete agent, role, team, workflow, ticket)
- Web UI: Undo action system with friendly labels (`index.html:4808`)
- Web UI: Login form inputs have `autofocus`, `autocomplete`, and `required` attributes (`index.html:941-942`)

### Issues found
| Finding | Severity | Location | Recommendation |
|---------|----------|----------|----------------|
| Web UI: no loading/disabled state on async action buttons | High | `index.html:3288, 5659` | Disable button + show "Deleting…"/"Saving…" during `await call()`; re-enable in `finally` |
| Form `<label>` elements not associated with inputs in ticket/project modals | High | `index.html:1209-1232, 1349+` | Use `<label for="input-id">` or wrap inputs; enables click-to-focus and screen readers |
| Web UI: no dark mode — hard-coded light palette, no `prefers-color-scheme` | Medium | `index.html` CSS `:root` block | Add `@media (prefers-color-scheme: dark)` overrides using existing CSS variable names |
| Web UI: no success toast after save/create — silent completion | Medium | Web UI generally | Add transient 2s status message: "Saved", "Ticket created", "Agent updated" etc. |
| Web UI: team member and project member removal has no confirmation | Medium | `index.html:3706, 3735, 4211, 4235` | Add `uiConfirm()` before removing users/agents from teams and projects |
| Web UI: workflow stage delete has no confirmation | Medium | `index.html:4045` | Add `uiConfirm()` before DELETE on workflow stage |
| CLI state differentiation is colour-only — inaccessible to colour-blind users | Medium | `cmd/ticket/printer.go` | Add symbol prefix alongside colour: `●` active, `✓` success, `✗` fail, `○` idle |
| Only 1 breakpoint at 900px — no 480px mobile or 1200px wide breakpoint | Medium | `index.html:308-312` | Add 480px breakpoint (stack columns, full-width panels) and 1200px wide layout |
| SVG pixel logos have no `alt` text or `aria-label` | Low | `index.html:938, 1550` | Add `aria-label="Ticket"` to `<canvas>` elements used as logos |
| Keyboard shortcuts not documented in web UI | Low | `index.html:5942-6072` | Add `?` shortcut to show keyboard help overlay |
| CLI `tk ticket help` does not list available verbs | Low | `cmd/ticket/help.go` | Include verb list (add, update, rm, assign, claim, complete…) in ticket namespace help |

## Verdict
Two significant gaps from the previous assessment are fixed: `tk rm` now has a two-step confirmation with a generated token (was Critical), and web UI ticket delete now has a `uiConfirm()` guard (was High). These are the most user-safety-relevant improvements. The remaining gaps are: missing loading states on buttons, unlinked form labels, no dark mode, silent success operations, and a handful of unconfirmed destructive actions (team member removal, workflow stage delete). Score improves from 64 to 68.

## Changes since last assessment
- **Fixed (Critical):** `tk rm` — two-step `--confirm <token>` pattern now implemented (`cmd_ticket.go:2263-2286`), matching project delete behaviour.
- **Fixed (High):** Web UI ticket delete — `uiConfirm("Delete ticket…?", "Delete")` now present before DELETE API call (`index.html:5657`).
- No other UX-relevant changes between v0.1.730 and v0.1.737.
