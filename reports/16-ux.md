# UX Review

**Score: 76/100** (was 74)

## What is being assessed

The single-page application at `web/static/index.html` (6,080 lines) across ten UX dimensions, with attention to SDLC management UI added in the refactor.

## Methodology

Read HTML/CSS/JS monolith. Checked CSS variables, media queries, ARIA attributes, modal patterns, keyboard handlers, form validation, confirm dialogs, error display, notifications, and new SDLC management view.

## Findings

### Passing checks
- Viewport meta correct — `index.html:5`
- Mobile panel breakpoint at 900px — `index.html:308-312`
- `min()` fluid sizing on login, modals, search
- Confirm dialogs on destructive actions — `uiConfirm()` at lines 3234, 3508, 3832, 4084, 5657, 5973
- Reusable dialog system — `openDialog()`/`uiAlert()`/`uiConfirm()`
- Per-form status feedback elements
- Keyboard navigation on board — double-Shift, N, V, Tab, Shift+Arrows, E/Enter, D, U
- 250ms debounced autosave on ticket form — `index.html:5191-5199`
- 50-action undo stack with `appStatus` feedback — `index.html:4805-4829`
- ARIA basics: `role="dialog"`, `aria-modal`, `aria-label`, `aria-expanded`
- Focus management on modal open
- Chat loading state with pending animation
- SDLC management view present with cards, create/delete/export/import
- Perspective fade animation on view transitions

### Issues found

| Finding | Severity | Location | Recommendation |
|---------|----------|----------|----------------|
| SDLC UI calls `/api/sdlc` (singular) — server uses `/api/sdlcs` — all SDLC ops return 404 | Critical | `index.html:3958-4154` vs `api_sdlc.go:19` | Replace `/api/sdlc` with `/api/sdlcs` (8 occurrences) |
| No SDLC selector on project create/update modal | High | `index.html:4955-4963` | Add SDLC dropdown, include `sdlc_id` in payload |
| Stage-add form has no role picker — stage-role assignment is CLI-only | High | `index.html:4103` | Add role picker |
| SDLC view has no card/list toggle unlike agents/roles/teams | High | `index.html:1078-1089` | Add toggle to match pattern |
| `#app-status` element is `display:none` — undo/error messages invisible | High | `index.html:917` | Make visible with styling |
| No loading spinners on async operations | Medium | Various | Add spinners/skeletons |
| No flash/toast notification system | Medium | General | Implement global toast |
| Delete buttons not visually distinct (no `.btn.danger` class) | Medium | `index.html:74-80` | Define and apply danger style |
| No dark-mode support for main app | Medium | `index.html:8-24` | Add `prefers-color-scheme` media query |
| Board columns not responsive below 900px | Medium | `index.html:343` | Add single-column stacking at 600px |
| Login inputs lack `<label>` elements | Medium | `index.html:941-942` | Add labels for accessibility |
| Ticket modal inputs lack `<label for>` | Low | `index.html:1357-1444` | Add label associations |
| Members view requires numeric IDs — no search/autocomplete | Low | `index.html:1105,1122` | Add name search |

## Verdict

SDLC management view skeleton added with consistent card pattern and undo system. Score +2 to 76, offset by the API URL bug making SDLC UI non-functional, and missing UI connection points for project-SDLC assignment and stage-role management.
