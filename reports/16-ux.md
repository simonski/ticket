# UX Review

**Score: 64/100**

## What is being assessed
CLI UX (help text, error messages, output formatting, destructive operation safety, discoverability, colour usage) and web UI UX (form validation, loading states, error display, mobile responsiveness, dark mode, keyboard navigation, accessibility, confirmation dialogs, success feedback).

## Methodology
Reviewed `cmd/ticket/help.go`, `cmd/ticket/printer.go`, `cmd/ticket/cmd_ticket.go`, `cmd/ticket/cmd_project.go`, and `web/static/index.html` (6077 lines). Checked for ARIA attributes, media queries, confirmation dialogs, loading state patterns, and colour-only state differentiation.

## Findings

### Passing checks
- CLI help system: hierarchical help with USAGE, DETAILS, EXAMPLE, ENVIRONMENT sections (`help.go`)
- Terminal-aware output: `term.GetSize()` with min-width clamping (`printer.go:383-392`)
- Consistent Unicode box-drawing tree rendering in `tk ls`
- Semantic colour palette: green=active, red=fail, gray=success, white=idle (`printer.go`)
- `--no-color` flag respected globally (`printer.go:271`)
- Project deletion has two-step confirmation with token (`cmd_project.go`)
- Web UI: WebSocket health monitoring with automatic reconnect polling (`index.html:4490`)
- Web UI: Viewport meta tag present â€” `width=device-width, initial-scale=1` (`index.html:5`)
- Web UI: Responsive grid with `auto-fit, minmax(220px, 1fr)` (`index.html:529`)
- Web UI: Extensive keyboard navigation (arrow keys, Tab, E/D/U/S/P shortcuts) (`index.html:5942-6072`)
- Web UI: Semantic HTML elements: `<header>`, `<main>`, `<aside>`, `<nav>`, `<section>`
- Web UI: ARIA attributes: `role="dialog"`, `aria-modal="true"`, `aria-expanded`, `aria-label` on nav/modals
- Web UI: `uiConfirm()` used for all destructive web operations (delete agent, role, team, workflow)
- Web UI: Undo action system with friendly labels (`index.html:4808`)

## Verdict
CLI UX is polished with good colour usage, terminal-aware output, and comprehensive help. The critical gap is missing delete confirmation for tickets in both CLI and web UI. The web UI has strong keyboard navigation and ARIA foundations but is let down by missing loading states, silent error handlers, and inaccessible form labels. Dark mode and mobile layout are secondary improvements.
